Docker & K8S (1)

September 18th, 2019

  • мҝ лІ„л„ӨнӢ°мҠӨлҠ” кө¬кёҖмқҙ м§ҖлӮң 10м—¬л…„ к°„ м»Ён…Ңмқҙл„ҲлҘј н”„лЎңлҚ•м…ҳ нҷҳкІҪм—җм„ң мӮ¬мҡ©н•ҳл©ҙм„ң м–»мқҖ кІҪн—ҳмқ„ л°”нғ•мңјлЎң к°ңл°ңн•ң м»Ён…Ңмқҙл„Ҳ-нҶөн•©лҸ„кө¬мқҙлӢӨ.
  • м—¬лҹ¬ к°ңмқҳ нҳёмҠӨнҠёлҘј н•ҳлӮҳлЎң л¬¶м–ҙ DockerлҘј мқҙмҡ©н•ҳкё° мң„н•ң мҳӨмјҖмҠӨнҠёл Ҳмқҙм…ҳ нҲҙ

л¶„мӮ° нҷҳкІҪм—җм„ң н•ң лҢҖмқҳ м»ҙн“Ён„°мІҳлҹј нҲ¬кіјм ҒмңјлЎң м»Ён…Ңмқҙл„Ҳм—җ м—‘м„ёмҠӨн•  мҲҳ мһҲлӢӨ.

мӢңмҠӨн…ң мқҙмҡ©мһҗлЎңл¶Җн„° мҳӨлҠ” л¶Җн•ҳмқҳ кёүмҰқм—җ лҢҖн•ҙ мҠӨмјҖмқјн•ҳкұ°лӮҳ м—¬лҹ¬ м»Ён…Ңмқҙл„ҲлҘј нҡЁмңЁм ҒмңјлЎң нҶөн•© кҙҖлҰ¬н•ҳлҠ” мһҘм№ҳлҸ„ мһҲлӢӨ.

мЈјмҡ” кё°лҠҘл“Ө

  • м—¬лҹ¬ м„ңлІ„л“Өм—җм„ңмқҳ м»Ён…Ңмқҙл„Ҳ кҙҖлҰ¬
  • м»Ён…Ңмқҙл„Ҳ к°„ л„ӨнҠёмӣҢнҒ¬ кҙҖлҰ¬
  • м»Ён…Ңмқҙл„Ҳмқҳ л¶Җн•ҳ л¶„мӮ°
  • м»Ён…Ңмқҙл„Ҳмқҳ к°җмӢң
  • л¬ҙм •м§ҖлЎң м—…лҚ°мқҙнҠё

Kubernetesмқҳ м„ңлІ„ кө¬м„ұ

Master Server

  • kubernetes нҒҙлҹ¬мҠӨн„° м•Ҳмқҳ м»Ён…Ңмқҙл„ҲлҘј мЎ°мһ‘н•ҳкё° мң„н•ң server

kubect11 лӘ…л №мқ„ мӮ¬мҡ©н•ҳм—¬ нҒҙлҹ¬мҠӨн„°лҘј кө¬м„ұн•ҳкұ°лӮҳ лҰ¬мҶҢмҠӨлҘј мЎ°мһ‘н•  л•Ң Master Serverк°Җ commandлЎңл¶Җн„° requestлҘј л°ӣм•„ мІҳлҰ¬н•ңлӢӨ. нҒҙлҹ¬мҠӨн„° лӮҙмқҳ л…ёл“ңмқҳ лҰ¬мҶҢмҠӨ мӮ¬мҡ© нҳ„нҷ©мқ„ нҷ•мқён•ҳкі , м»Ён…Ңмқҙл„ҲлҘј мӢңмһ‘н•  л…ёл“ңлҘј мһҗлҸҷмңјлЎң м„ нғқн•ңлӢӨ. лӢӨмӨ‘нҷ”лҘј кІҖнҶ н•  н•„мҡ”к°Җ мһҲлӢӨ.

Back-end Database(etcd)

  • л¶„мӮ° key-value store, нҒҙлҹ¬мҠӨн„°мқҳ кө¬м„ұ м •ліҙлҘј кҙҖлҰ¬н•ңлӢӨ.

нҒҙлҹ¬мҠӨн„°лҘј кө¬м¶•н•ҳкё° мң„н•ң м„Өм • м •ліҙк°Җ л“Өм–ҙ мһҲлӢӨ.

мӢңмҠӨн…ң кө¬м„ұм—җ л”°лқјм„ң etcdлҘј master serverмғҒм—җ кө¬м¶•н•ҳлҠ” кІҪмҡ°лҸ„ мһҲлӢӨ. master serverмҷҖ л§Ҳм°¬к°Җм§ҖлЎң лӢӨмӨ‘нҷ”лҘј кІҖнҶ н•  н•„мҡ”к°Җ мһҲлӢӨ.

Node

  • мӢӨм ңлЎң Docker containerлҘј мһ‘лҸҷмӢңнӮӨлҠ” м„ңлІ„

л…ёл“ңлҘј м—¬лҹ¬ к°ң л§Ҳл Ён•ҳм—¬ clusterлҘј кө¬м„ұн•ңлӢӨ. master serverмқҳ кҙҖлҰ¬лҘј л°ӣлҠ”лӢӨ. л…ёл“ңлҘј лӘҮ лҢҖ л§Ҳл Ён• м§ҖлҠ” мӢңмҠӨн…ңмқҳ к·ңлӘЁлӮҳ л¶Җн•ҳм—җ л”°лқј лӢ¬лқјм§„лӢӨ. нҒҙлқјмҡ°л“ң нҷҳкІҪм—җм„ңлҠ” Node н•ҳлӮҳк°Җ к°ҖмғҒ лЁёмӢ мқҳ мқёмҠӨн„ҙмҠӨ н•ҳлӮҳк°Җ лҗңлӢӨ.

Application кө¬м„ұ кҙҖлҰ¬

application мӢӨн–ү нҷҳкІҪмқҳ orchestration мқ„ мң м—°н•ҳкІҢ мҲҳн–үн•ҳкё° мң„н•ҙ лӢӨм–‘н•ң м¶”мғҒнҷ”лҘј н•ҳкі  мһҲлӢӨ.

Pod

  • м—¬лҹ¬к°ңмқҳ м»Ён…Ңмқҙл„ҲлҘј лӘЁм•„м„ң 'pod'лЎң кҙҖлҰ¬н•ңлӢӨ.

app serverмҡ© м»Ён…Ңмқҙл„ҲмҷҖ proxy serverмҡ© м»Ён…Ңмқҙл„Ҳ л“ұ кҙҖл Ёлҗң кІғл“Өмқ„ podлЎң лӘЁмқҖлӢӨ. PodлҠ” applicationмқҳ Deployment лӢЁмң„к°Җ лҗҳл©° Pod лӢЁмң„лЎң м»Ён…Ңмқҙл„Ҳ мһ‘м„ұ/мӢңмһ‘/м •м§Җ/мӮӯм ң мҷҖ к°ҷмқҖ мЎ°мһ‘мқ„ мҲҳн–үн•ңлӢӨ. л”°лқјм„ң мӣ№ н”„лЎ нҠё м„ңлІ„мҷҖ лҚ°мқҙн„°лІ мқҙмҠӨ м„ңлІ„мІҳлҹј лӢӨлҘё кё°лҠҘмқ„ н•ҳлӮҳмқҳ Podм—җ м ҖмһҘн•ҳл©ҙ м•ҲлҗңлӢӨ.

PodлҠ” л°ҳл“ңмӢң лҸҷмқјн•ң л…ёл“ң мғҒм—җ лҸҷмӢңм—җ м „к°ңлҗңлӢӨ. Pod м•Ҳмқҳ м—¬лҹ¬ м»Ён…Ңмқҙл„Ҳк°Җ к°ҖмғҒ NIC(private IP)2лҘј кіөмң н•ҳлҠ” кө¬м„ұмқ„ м·Ён•ҳкё° л•Ңл¬ём—җ м»Ён…Ңмқҙл„ҲлҒјлҰ¬ localhost кІҪмң лЎң нҶөмӢ н•  мҲҳ мһҲлӢӨ.

Replica Set

  • нҒҙлҹ¬мҠӨн„° мғҒм—җ лҜёлҰ¬ м§Җм •лҗң PodлҘј мһ‘м„ұн•ҳм—¬ мӢӨн–үмӢңмјңл‘җлҠ” мһҘм№ҳ

нҒҙлҹ¬мҠӨн„° мғҒм—җ м •н•ҙм§„ мҲҳмқҳ PodлҘј л°ҳл“ңмӢң мӢӨн–үмӢңмјң л‘”лӢӨлҠ” кІғ мӢӨн–үмӨ‘мқё PodлҘј к°җмӢңн•ҳкі  м •м§Җлҗң кІҪмҡ°м—җ н•ҙлӢ№ PodлҘј мӮӯм ңн•ҳкі , мғҲлЎңмҡҙ PodлҘј мӢӨн–үмӢңнӮЁлӢӨ. нҒҙлқјмҡ°л“ң м•Ҳм—җ podлҘј м–јл§ҲлӮҳ мӢӨн–үмӢңмјң л‘ҳм§ҖлҘј 'Replica мҲҳ'лқјкі  н•ңлӢӨ. podмқҳ мҲҳлҘј лҸҷм ҒмңјлЎң ліҖкІҪн•ҳм—¬ мҳӨнҶ мҠӨмјҖмқјмқ„ кө¬нҳ„н•  мҲҳ мһҲлӢӨ.

Deployment

  • PodмҷҖ ReplicaSetмқ„ лӘЁмқҖ кІғ ReplicaSetмқҳ мқҙл Ҙмқ„ кҙҖлҰ¬н•ҳлҠ” кІғмқҙлӢӨ.

ReplicaSetмқҳ н…ңн”ҢлҰҝмқ„ к°Җм§Җкі  кұ°кё°м„ң Podмқҳ кө¬м„ұмқ„ м •мқҳн•ҳм—¬ н•ҙлӢ№ н…ңн”ҢлҰҝмқ„ л”°лҘҙлҠ” ReplicaSetмқ„ л§Ңл“ лӢӨ. мқҙл Ҙмқ„ кҙҖлҰ¬н•  мҲҳ мһҲкё° л•Ңл¬ём—җ pod м•Ҳмқҳ м»Ён…Ңмқҙл„Ҳмқҳ лІ„м „м—…мқ„ н•ҳкі  мӢ¶мқ„ л•Ң лЎӨл§Ғ м—…лҚ°мқҙнҠё3лҘј н•ҳкұ°лӮҳ мқҙм „ м„ёлҢҖлЎң лЎӨл°ұн•  мҲҳ мһҲлӢӨ. кё°ліём ҒмңјлЎң 'Blue-Green Deployment' л°©лІ•мқҙ мһҲлӢӨ.

мқҙ мҷём—җлҸ„

  • DeamonSet : Nodeлі„лЎң к°җмӢң м—җмқҙм „нҠёмҷҖ к°ҷмқҖ нҠ№м • podлҘј л°ҳл“ңмӢң л°°м№ҳн•ҳкі  мӢ¶мқ„ л•Ң мӮ¬мҡ©
  • Jobs : мғҒмЈј м„ңл№„мҠӨк°Җ м•„лӢҢ мҲҳм№ҳм—°мӮ° мІҳлҰ¬мҷҖ к°ҷмқҙ н”„лЎңк·ёлһЁмқҳ мӢңмһ‘л¶Җн„° мў…лЈҢк№Ңм§ҖлЎң мҷ„лЈҢлҗҳлҠ” н”„лЎңк·ёлһЁмқ„ Podм—җм„ң мӢӨн–үмӢңнӮЁлӢӨ.
  • CronJob : Linuxмқҳ cron4кіј к°ҷмқҙ PodлҘј мӢӨн–үмӢңнӮ¬ нғҖмқҙл°Қмқ„ м§Җм •н•ңлӢӨ.

л“ұмқҳ кҙҖлҰ¬ нҲҙл“Өмқҙ мһҲлӢӨ.

Network кҙҖлҰ¬(Service)

нҒҙлҹ¬мҠӨн„° м•Ҳм—җм„ң мӢӨн–үлҗҳлҠ” Podм—җ лҢҖн•ҙ мҷёл¶ҖлЎңл¶Җн„° м—‘м„ёмҠӨн•  л•ҢлҠ” м„ңл№„мҠӨлҘј м •мқҳн•ңлӢӨ. ServiceлҠ” л„ӨнҠёмӣҢнҒ¬лҘј кҙҖлҰ¬н•ҳлҠ” кІғмңјлЎң мў…лҘҳк°Җ лӢӨм–‘н•ҳлӢӨ.

  • Load Balancer : Serviceм—җ лҢҖмқ‘н•ҳлҠ” IP мЈјмҶҢ + нҸ¬нҠёлІҲнҳём—җ м—‘м„ёмҠӨн•ҳл©ҙ м—¬лҹ¬ Podм—җ лҢҖн•ң L4 levelмқҳ лЎңл“ңлІЁлҹ°мӢұмқҙ мқјм–ҙлӮңлӢӨ.

Serviceм—җ н• лӢ№лҗҳлҠ” IP мЈјмҶҢлҠ”

  • Cluster IP : cluster лӮҙл¶Җмқҳ PodлҒјлҰ¬мқҳ нҶөмӢ мқ„ мң„н•ң Private IP podм—җм„ң ліҙлӮҙлҠ” нҢЁнӮ·мқҖ Node мғҒмқҳ Proxy лҚ°лӘ¬мқҙ л°ӣм•„ мҲҳмӢ  PodлЎң м „мҶЎлҗңлӢӨ.
  • External IP : мҷёл¶Җ clientмҷҖ м—°кІ°н•ҳкё° мң„н•ң public IP мЈјмҶҢ PodлҘј мғҲлЎң мӢӨн–үн•ҳл©ҙ кё°мЎҙ м„ңл№„мҠӨмқҳ IP мЈјмҶҢмҷҖ нҸ¬нҠё лІҲнҳёлҠ” нҷҳкІҪліҖмҲҳлЎң м°ёмЎ°н•  мҲҳ мһҲлӢӨ.

Service мҷём—җ..

IngressлқјлҠ” podм—җ лҢҖн•ң нҶөмӢ мқ„ м ңм–ҙн•ҳлҠ” кё°лҠҘмқҙ мһҲлӢӨ. IngressлҠ” ServiceмҷҖ м—°кІ°лҗҳм–ҙ нҶөмӢ  лӮҙмҡ©мқ„ мӨ‘кі„(proxy)н•ңлӢӨ. мқҙлҠ” мһ‘лҸҷн•ҳлҠ” нҷҳкІҪм—җ л”°лқј лӢӨлҘёлҚ°, GCPмқҳ кІҪмҡ° HTTP Load BalancerлҘј мӮ¬мҡ©н•ңлӢӨ. L7м—җм„ң мһ‘лҸҷн•ҳкё° л•Ңл¬ём—җ request url лі„ л¶„лҘҳлӮҳ virtual host кё°лҠҘ л“ұ ліҙлӢӨ м„ём„ён•ң л„ӨнҠёмӣҢнҒ¬ м ңм–ҙлҘј н•  мҲҳ мһҲлӢӨ.

Labelмқ„ мӮ¬мҡ©н•ң лҰ¬мҶҢмҠӨ мӢқлі„

лҰ¬мҶҢмҠӨлҘј мӢқлі„н•ҳкё° мң„н•ҙ лӮҙл¶Җм—җм„ң мһҗлҸҷмңјлЎң лһңлҚӨн•ң мқҙлҰ„мқҙ л¶Җм—¬лҗЁ мқҙ мқҙлҰ„мңјлЎң м Ғм ҲнһҲ кҙҖлҰ¬н•ҳкё° м–ҙл өкё° л•Ңл¬ём—җ Labelмқ„ л¶ҷм—¬м„ң кҙҖлҰ¬н•ңлӢӨ.

  • лҰ¬мҶҢмҠӨлҘј мӢқлі„н•ҳкё° мң„н•ң Key-Value нҳ•нғңмқҳ мһ„мқҳмқҳ л¬ёмһҗм—ҙ

Labelмқ„ identifierлЎң н•ҳм—¬ лҰ¬мҶҢмҠӨлҘј мқјкҙ„м ҒмңјлЎң мІҳлҰ¬н•ңлӢӨ.

LabelмқҖ н•ҳлӮҳмқҳ лҰ¬мҶҢмҠӨм—җ м—¬лҹ¬к°ңлҘј м„Өм •н•  мҲҳ мһҲм–ҙм„ң Podмқҳ м—ӯн• лі„лЎң мһ„мқҳмқҳ мқҙлҰ„мқ„ л¶ҷмқҙкұ°лӮҳ кҙҖл Ё мһҲлҠ” Pod лі„лЎң лӘЁм•„м„ң мң м—°н•ҳкІҢ кҙҖлҰ¬н•ҳкі мһҗ н• л•Ң мң мҡ©н•ҳлӢӨ.

Kubernetesмқҳ кө¬мЎ°

    componentл“ӨмқҖ etcdм—җ м ‘к·јн•ҳкё°мң„н•ҙ API serverлҘј кұ°м№ҳкі 

    н”„лЎңк·ёлһҳлЁёлҠ” мӣ№мқҳ GUIнҲҙмқҙлӮҳ kubectl лӘ…л №мқ„ мӮ¬мҡ©н•ҳкі , app лӮҙм—җм„ң API ServerлҘј нҳём¶ңн•ңлӢӨ.

    мқёмҰқ, мқёк°Җ кё°лҠҘмқ„ к°–кі  мһҲмқҢ
